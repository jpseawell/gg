# docs

The `docs` folder is the single place for non-runtime project context.

## Sections

- `product` - requirements, goals, roadmap, audience, and scope notes
- `technical` - architecture notes, ADRs, diagrams, and implementation constraints
- `agentic` - prompts, workflow notes, and repository-specific automation rules
- `backlog` - prioritized work and planning snapshots
- `runbooks` - operational procedures and incident handling notes
- `templates` - reusable document starters for repeatable work
