# Security

## Supported Versions

This template tracks the default branch. Projects created from it should define their own supported release policy once they have production users.

## Reporting a Vulnerability

Use GitHub private vulnerability reporting if it is enabled for the repository. If it is not enabled, contact the repository owner directly instead of opening a public issue.

Include:

- affected package or workspace
- reproduction steps
- expected impact
- relevant logs, screenshots, or proof-of-concept details

Do not include secrets, private keys, production customer data, or credentials in reports.

## Dependency Review

Run dependency audits from an environment where sending package metadata to the registry audit service is acceptable for the project.
